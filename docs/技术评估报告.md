# TimeLoop 技术评估报告（面向课程答辩/提交）

## 1. 项目定位与目标

`TimeLoop: The Unspeakable Midnight` 是一个基于 LLM 的克苏鲁风格时间循环互动叙事系统。项目目标并非仅“生成文本”，而是实现一条完整 NLP 管线：

- 输入理解（NLU）：识别玩家意图与关键信息；
- 叙事生成（NLG）：基于世界状态与角色关系生成剧情；
- 状态演化：在回合内与跨循环层面维护一致的游戏状态；
- 一致性约束：通过硬规则/软约束减少幻觉与逻辑冲突；
- 可评估性：通过日志与脚本化评测给出可复现指标。

该定位符合课程项目中“问题定义明确、方法闭环、可量化验证”的要求。

## 2. 系统架构与实现完整性

### 2.1 核心架构

系统采用流水线处理：`PreCheck -> Event -> Trust -> KnowledgePre -> LLM -> KnowledgePost -> PostEvent -> Ending`。  
其中关键模块职责清晰：

- `src/game/engine.py`：总控流程、事件应用、回合结果聚合；
- `src/game/event_system.py`：事件触发条件判定与时间线；
- `src/state/*`：`GameState`（单循环）+ `LoopMemory`（跨循环）；
- `src/llm/*`：提示词构建、结构化输出解析、模型调用；
- `src/consistency/*`：硬规则校验与软一致性检查；
- `src/ui/app.py`：Gradio 前端呈现与交互。

### 2.2 数据与知识建模

根据 `eval/DATA_CARD.md`：

- 事件脚本 29 条（双语，含触发条件、效果、选项）；
- 地点 6 个、交互 NPC 4 个、事实 18 条；
- 规则 10 条（其中 5 条有 Python 硬实现）；
- 结局 3 条，分支决策点 4 个。

数据结构化程度较高，支持条件触发、状态更新与回合回放分析，满足课程对“可计算语义表示”的基本要求。

## 3. 评测结果解读（基于 FINAL_EVALUATION_REPORT）

### 3.1 NLU 表现

- 总评测 219 回合，`UNKNOWN` 意图占比 3.7%；
- 主意图（`TALK/INVESTIGATE/EXPLORE`）占比 88.1%；
- 自由输入占比 8.2%，说明系统并非只依赖按钮驱动。

评估：在开放文本输入场景下，3.7% 未识别率属于可接受区间，但仍有模糊中文输入的识别稳定性空间。

### 3.2 NLG 与上下文一致性

- 共生成叙事 219 条，支持中英双语；
- 叙事可体现地点、理智值、NPC 关系等上下文变量；
- 案例显示在低理智状态下风格可控变化（distorted）。

评估：NLG 已从“通用续写”走向“状态条件生成”，技术含量高于纯 prompt demo。

### 3.3 状态追踪与规则稳定性

- 共记录 172 次状态更新（平均 0.79 次/回合）；
- 最终输出中硬规则违规率报告为 0.0%（0/219）；
- 跨循环记忆、信任衰减、时间推进均有运行证据。

评估：项目最强项是“生成 + 规则 + 状态机”的组合，显著提升了系统 soundness。

### 3.4 延迟与交互体验

- P50 延迟约 15.7s，P95 约 46.9s；
- 主要瓶颈在 LLM API 网络与重试开销。

评估：以回合制文本游戏标准看“可玩”，但若作为产品仍需优化推理时延。

## 4. 对“老师要求/评分导向”的对齐情况

结合课程报告蓝图（`eval/REPORT_BLUEPRINT.md`）与现有实现，项目在关键维度的对齐如下：

### 4.1 Soundness（方法可靠性）

- 明确 I/O：输入（文本/选择）-> 结构化 JSON 输出；
- 规则闭环：硬规则检查 + 失败重试机制；
- 状态闭环：单循环状态与跨循环记忆分层管理。

结论：对齐较好，是当前项目得分潜力最高维度。

### 4.2 Reproducibility（可复现）

- 配置集中在 `config/settings.yaml`；
- 轨迹日志与评测报告文件完整；
- 数据源和脚本路径清晰（`eval/*`）。

结论：基础可复现性达标，建议补充一键评测命令说明与随机种子记录。

### 4.3 Evaluation Rigor（评估严谨性）

- 已有量化指标：意图分布、违规率、延迟、分支多样性；
- 已有案例分析：成功/失败模式均覆盖；
- 仍缺：人工主观评价（如叙事质量 MOS）与更大规模 A/B 对照。

结论：具备“工程评测”深度，尚可加强“学术评测”广度。

## 5. 主要风险与改进优先级

### 高优先级（建议先做）

1. **意图集合约束收紧**：减少越界意图标签（如 READ/FLEE 等）；
2. **输出解析稳健性**：继续强化 JSON 修复与回退路径；
3. **端到端时延优化**：缓存更强、上下文裁剪更激进、可选小模型路由。

### 中优先级

1. 补齐硬规则实现覆盖（目前 10 定义 / 5 强实现）；
2. 增加跨循环“当前轮次已知事实”与“历史已知事实”分离，降低时序误导；
3. 扩展到多剧情世界，验证框架泛化能力。

## 6. 总体结论

从课程项目视角，`TimeLoop` 已达到“可展示、可解释、可评估”的完整系统标准，核心亮点在于：

- 不是单点模型调用，而是可控叙事系统；
- 具有较强一致性保障和跨循环机制设计；
- 有较充分的量化证据支持技术主张。

若后续在“时延优化 + 评测方法扩展 + 规则完备性”三方面继续迭代，项目可进一步提升到接近研究型课程项目的完成度。
