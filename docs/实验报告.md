# TimeLoop 实验报告

## 1. Experimental Setup

本实验聚焦于验证系统在互动叙事任务中的四类核心能力：输入理解（NLU）、上下文生成（NLG）、一致性维护、交互响应性能。  
数据与结果基于真实游玩日志统计，而非合成样本。

### 1.1 实验配置

| 项目 | 配置 |
|---|---|
| 项目 | TimeLoop: The Unspeakable Midnight |
| 任务形态 | 互动文本冒险（回合制，多分支） |
| 评测数据来源 | `eval/results/FINAL_EVALUATION_REPORT.md` 对应日志统计 |
| 会话规模 | 3 个 gameplay sessions |
| 循环规模 | 6 个 time loops |
| 总回合数 | 219 turns |
| 模型 | Qwen/Qwen2.5-7B-Instruct |
| API Provider | SiliconFlow（OpenAI-compatible） |
| 生成参数 | temperature=0.7, max_tokens=1024 |
| 输出格式 | Structured JSON mode |
| 一致性机制 | HardRulesChecker（5个已实现硬规则）+ Soft checker（dev） |
| Prompt 策略 | 3-tier（Static System / Dynamic Context / Turn Payload） |
| 运行环境 | 本地 Windows + Python + Gradio（详版可在最终报告补充版本号） |

### 1.2 评测范围说明

- 本次评测覆盖了按钮驱动输入与自由文本输入两类交互；
- 含多轮跨循环行为，可观察记忆系统与策略迁移；
- 包含成功与失败样例，便于分析系统鲁棒性。

## 2. Metric Definitions

为保证可复现与可比较，所有指标给出明确计算定义、单位与样本范围。

### M1. Intent Coverage（意图覆盖率）

定义：

\[
\text{Intent Coverage} = 1 - \frac{\#UNKNOWN}{N_{turns}}
\]

- 含义：越高表示 NLU 将输入映射到有效意图的能力越强；
- 单位：%；
- 样本：全部回合（\(N_{turns}=219\)）。

### M2. Final Consistency Violation Rate（最终输出一致性违规率）

定义：

\[
\text{Violation Rate} = \frac{\#HardRuleViolations_{final}}{N_{turns}}
\]

- 含义：最终给到玩家的输出中，违反硬规则的比例；
- 单位：%；
- 样本：全部回合（219）。

### M3. State Update Density（状态更新密度）

定义：

\[
\text{Update Density} = \frac{N_{state\_updates}}{N_{turns}}
\]

- 含义：每回合平均状态变更次数，反映“交互是否真正驱动世界状态变化”；
- 单位：updates/turn；
- 样本：全部回合（219）。

### M4. Latency Distribution（延迟分布）

统计：

- Mean（均值）
- P50（中位数）
- P95（高分位延迟）

- 含义：反映实时交互可用性；
- 单位：秒（s）；
- 样本：有延迟记录回合（\(N=153\)）。

### M5. Branching Diversity（分支多样性）

定义：

\[
\text{Branching Diversity} = \frac{\#UniqueLocationSequences}{\#Loops}
\]

- 含义：不同循环的路径分化程度；
- 单位：%；
- 样本：6 loops。

## 4. Case Studies（标准模板）

以下案例按统一格式呈现：`Input / State / Output / Issue / Fix`。

### Case A（Success）：自由输入触发正确意图 + 上下文叙事

**Input**  
`"Scan the room for details while sipping tea."`

**State**  
Loop 1, Turn 6, Location=inn, Sanity=100（lucid）

**Output**  
- Intent = `INVESTIGATE`  
- 叙事内容与旅馆场景一致，并结合“喝茶”上下文  
- 触发 `update_trust: Martha +5`

**Issue**  
无（成功案例）

**Fix / Why it works**  
- 单次结构化输出同时完成 NLU 与 NLG；
- Prompt 中地点、NPC 与近期状态上下文被正确利用。

### Case B（Failure）：JSON 解析失败导致回退叙事

**Input**  
`"仔细查看这些纸张。"`

**State**  
Loop 1, Turn 21, Location=lighthouse, Sanity=67（uneasy）

**Output**  
- Intent = `UNKNOWN`  
- 输出回退到 fallback narration（非期望主路径）  
- 无有效 state update

**Issue**  
模型结构化 JSON 输出损坏（可能截断/格式不完整），解析失败后进入回退分支。

**Fix**  
- 增强 `_try_repair_truncated_json`；
- 增加 `_extract_narration_from_raw` 作为最后兜底；
- 调整 token 预算与上下文裁剪策略，降低超长上下文导致的格式损坏风险。

## 5. Analysis（面向评分项的结论）

### 5.1 Soundness（方法有效性）

- 指标定义明确且可复算，满足“data -> method -> system -> evaluation”闭环；
- 0.0% 最终硬规则违规率表明一致性防线有效；
- 成功与失败案例均有，且失败可解释、有对应修复路径。

### 5.2 系统权衡（质量 vs. 实时性）

- 一致性检查与重试机制显著提高稳定性，但对 P95 延迟有抬升；
- 现阶段 P50（15.7s）可用于回合制互动，P95（46.9s）仍是体验瓶颈；
- 后续优化方向：上下文预算控制、模型路由、重试策略更细粒度化。

### 5.3 有效性威胁（Threats to Validity）

- 样本规模有限（219 turns）；
- 单模型评测，跨模型泛化尚未验证；
- 延迟受网络波动影响，非纯模型推理耗时；
- 缺少人工主观打分（如 MOS）来补充叙事质量评价。
